## プログラミング記事投稿サイト

### 日本の主要プラットフォーム

#### 1. Qiita
- **URL**: https://qiita.com/
- **特徴**:
  - エンジニア向けの老舗プラットフォーム
  - プログラミングの知識、ノウハウを記録・共有
  - 幅広い技術情報を扱う
  - 多くのエンジニアに情報を届けやすい
- **制限**:
  - 収益化機能なし
  - 記事投稿のみ対応

#### 2. Zenn
- **URL**: https://zenn.dev/
- **特徴**:
  - 2020年リリースの比較的新しいプラットフォーム
  - 洗練されたUI/UX
  - **収益化可能**: 有料記事、Book販売、投げ銭
  - GitHub連携が標準機能
  - 記事・Book・スクラップの3つの投稿形式
  - Markdownに完全対応
  - コンテンツの所有権を保持
- **おすすめ用途**:
  - 専門知識を深掘りしたい人向け
  - 収益化を目指すエンジニア

#### 3. note
- **URL**: https://note.com/
- **特徴**:
  - エンジニア専門ではないが技術記事も投稿可能
  - 幅広い読者層にリーチしたい場合に有効

---

### 海外の主要プラットフォーム

#### 1. DEV.to
- **URL**: https://dev.to/
- **強み**:
  - トラフィック獲得が最も容易
  - すべての投稿が平等に「フィーチャー」される仕組み
  - 活発なコミュニティ
  - Markdown完全対応
- **制限**:
  - カスタムドメインやサブドメインでの独自ブログ不可
  - プラットフォーム上での投稿のみ

#### 2. Hashnode
- **URL**: https://hashnode.com/
- **強み**:
  - **無料でカスタムドメイン使用可能**
  - SEO対策が充実
  - コンテンツの完全な所有権
  - Markdown、コードスニペット、シンタックスハイライト対応
- **課題**:
  - トラフィック獲得が難しい
  - フィードでの露出が少ない傾向

#### 3. Medium
- **URL**: https://medium.com/
- **特徴**:
  - 全ジャンル対応の出版プラットフォーム
  - パートナープログラムによる収益化
- **課題**:
  - トラフィック獲得が困難（大手ブログやパートナープログラム参加者が優遇される）
  - Markdown非対応
  - コードスニペットやテーブルのサポートが弱い
  - 技術記事には不向き

#### 4. HackerNoon
- **URL**: https://hackernoon.com/
- **特徴**:
  - 開発者向けの投稿プラットフォーム
  - DEV.toやHashnodeと比較されることが多い

## MCP (Model Context Protocol) サーバー

### MCPとは

Model Context Protocol (MCP) は、Anthropicが2024年11月に発表したオープン標準・オープンソースフレームワークです。AIシステム（LLM）が外部ツール、システム、データソースと統合・共有する方法を標準化します。

**通称**: 「AIのUSB-C」

---

### 主要カテゴリー別MCPサーバー

#### 🔧 開発・バージョン管理系

1. **GitHub MCP** ⭐ 最人気
   - リポジトリ操作、Issue作成、PR管理を自然言語で実行
   - 開発者の生産性を劇的に向上
   - コードの読み取り、レビュー、リポジトリ設定管理

2. **Git MCP**
   - リポジトリの読み取り、検索、操作
   - Anthropic公式実装

3. **Filesystem MCP** ⭐ 定番
   - ファイルの読み書き・削除・一覧取得
   - セキュアなファイル操作、アクセス制御設定可能
   - MCPの基本として頻繁に使用される

---

#### 💾 データベース系

4. **PostgreSQL MCP**
   - クエリ実行、スキーマ検査
   - Anthropic公式実装
   - データベース接続、テーブル情報取得

5. **Supabase MCP** ⭐ コミュニティ人気No.1
   - 「型の不整合や幻覚が激減する」と高評価
   - DBへの読み取りアクセス提供

6. **MySQL MCP**
   - テーブル情報取得、クエリ実行
   - MySQLデータベースとの接続

---

#### 💬 コミュニケーション系

7. **Slack MCP** ⭐
   - メッセージ送信、ユーザープロフィール取得
   - チャンネル管理、リアクション機能
   - リアルタイムメッセージ投稿
   - 複数のトランスポート対応（Stdio、SSE）

8. **Gmail MCP**
   - OAuth2認証対応
   - メール送受信、検索、ラベル管理
   - 一括処理機能

9. **Discord MCP**
   - Bot経由でメッセージ送信
   - 過去メッセージ取得
   - 特定チャンネルへの自動投稿

---

#### ☁️ クラウド・ストレージ系

10. **Google Drive MCP** ⭐
    - ファイル操作、検索、メタデータ取得
    - Anthropic公式実装
    - クラウドストレージへの効率的なアクセス

11. **AWS MCP Servers** ⭐
    - AWSドキュメント、ベストプラクティスへのアクセス
    - AWS公式リリース
    - コンテキストガイダンス提供

---

#### 🔍 検索・AI系

12. **Perplexity Ask MCP**
    - Sonar APIでライブWeb検索
    - 公式MCPサーバー
    - AIエージェントがWeb検索を実行

13. **Fetch MCP**
    - Webコンテンツ取得・変換
    - Anthropic公式実装
    - HTMLをMarkdownに変換

---

#### 🎨 デザイン・その他

14. **Figma MCP**
    - デザインファイル連携
    - コンポーネント分析、コード変換
    - デザイン情報の正確な抽出

15. **Google Maps MCP**
    - 位置情報、経路案内、場所の詳細
    - ナビゲーション機能

16. **Spotify MCP**
    - 音楽プレイヤーの自然言語操作
    - プレイリスト管理

---

#### 🧪 開発補助系

17. **Puppeteer MCP**
    - ブラウザ自動操作
    - Anthropic公式実装
    - Webスクレイピング、テスト自動化

18. **Sequential Thinking MCP**
    - 構造的問題解決
    - Anthropic公式実装
    - 複雑なタスクの分解と実行

### 2025年の注目トレンド

#### 主要プラットフォームの対応状況

- **2024年11月**: Anthropic が MCP を発表
- **2025年3月**: OpenAI が MCP対応を公式発表
  - ChatGPTデスクトップアプリ
  - OpenAI Agents SDK
  - Responses API
- **2025年4月**: Google DeepMind が Gemini での MCP対応を発表
- **2025年時点**: コミュニティ貢献により1,000以上のMCPサーバーが公開

#### エコシステムの成長

- IDE、コーディングプラットフォーム（Replit）、コードインテリジェンスツール（Sourcegraph）がMCPを採用
- AIコーディングアシスタントにリアルタイムでプロジェクトコンテキストへのアクセスを提供
- エンタープライズシステムとの統合が進行中

---

### MCPの主な利用ケース

1. **コンテンツ作成**: FigmaのノートをMCP経由で取得し、AIライターがブログを編集
2. **開発支援**: Claude Desktop、Cursorなどの開発環境で統合
3. **データアクセス**: エンタープライズシステム（Google Drive、Slack、GitHub、Postgres、Stripe）への統合
4. **ワークフロー自動化**: 複数のツールとサービスを連携

---

### 公式実装 (Reference Servers)

Anthropic が提供する公式のリファレンス実装：

- **Everything**: プロンプト、リソース、ツールを含むリファレンス/テストサーバー
- **Fetch**: Webコンテンツの取得と変換
- **Filesystem**: セキュアなファイル操作、アクセス制御設定可能
- **Git**: Gitリポジトリの読み取り、検索、操作
- **PostgreSQL**: データベース接続とクエリ実行
- **Puppeteer**: ブラウザ自動化
- **Sequential Thinking**: 構造的問題解決
