# Claude Code 最新機能レポート (2025年版)

## 概要

Claude Codeは、AnthropicによるAI駆動の開発支援ツールです。2025年9月に大規模なアップデートが行われ、より自律的で強力な開発環境が提供されるようになりました。本レポートでは、Claude Codeの最新機能と特徴について包括的に解説します。

## 主要アップデート (2025年9月〜11月)

### Claude Code 2.0の主要機能

2025年9月、Anthropicは以下の主要なアップグレードを発表しました：

- **ネイティブVS Code拡張機能**（ベータ版）
- **ターミナルインターフェース2.0**
- **自律動作のためのチェックポイント機能**
- **Claude Sonnet 4.5をデフォルトモデルに採用**

---

## コア機能詳細

### 1. チェックポイント (Checkpoints)

**概要：** コードの状態と会話のコンテキストを自動保存し、いつでも以前の状態に戻せる機能です。

**主な特徴：**
- 各変更前に自動的にプロジェクト状態をスナップショット
- **Escキーを2回押す**、または`/rewind`コマンドで即座に復元
- Gitよりも強力：コードだけでなく、会話のコンテキストも巻き戻し可能
- AIエージェントが誤った方向に進んだ場合の「汚染された」会話状態もクリア可能

**利点：**
開発者の不安を直接的に解消し、AIによる変更を安心して試すことができます。従来のバージョン管理システムでは対応できなかった「会話の状態」も含めた完全な復元が可能です。

### 2. サブエージェント (Subagents)

**概要：** 専門的なタスクを委譲し、並列開発ワークフローを実現する機能です。

**主な特徴：**
- 複数のエージェントが同時に異なるタスクを処理
- 例：フロントエンド構築中にバックエンドAPIを別エージェントが構築
- カスタムサブエージェントの作成が可能

**使用例：**
```bash
# メインエージェントがフロントエンドを構築
# サブエージェントがバックエンドAPIを並行して構築
```

### 3. VS Code拡張機能

**概要：** VS Code内でClaude Codeを直接利用できるネイティブ拡張機能（ベータ版）です。

**主な機能：**
- リアルタイムで変更内容をサイドバーパネルに表示
- `@メンション`機能でファイルを参照
- ファイルや画像の添付
- MCPサーバーの利用
- 会話履歴の管理
- 複数セッションのサポート
- キーボードショートカット
- ほとんどのCLIスラッシュコマンドをサポート
- 拡張思考モード（Extended Thinking）のトグル

**インストール：**
VS Code Extension Marketplaceからダウンロード可能です。

**注意事項：**
- サブエージェントの設定はCLI経由で行う必要があります
- チェックポイント機能はVS Codeでも利用可能

### 4. ターミナルインターフェース 2.0

**改善点：**
- **検索可能なプロンプト履歴**（Ctrl+r）
- 以前のプロンプトの再利用や編集が容易に
- ステータス表示の改善
- より直感的なUX

### 5. フック (Hooks)

**概要：** ワークフローの重要なポイントで自動的にアクションをトリガーする機能です。

**主な用途：**
- コード変更後に自動的にテストスイートを実行
- コミット前の自動リント処理
- セッション終了時のアクティビティログ記録
- 品質ゲートの自動実施

**特徴：**
- 品質基準の自動実施
- 特定イベントへの自動反応
- コードの自動フォーマット

### 6. バックグラウンドタスク (Background Tasks)

**概要：** 長時間実行されるプロセスを他の作業をブロックせずに維持する機能です。

**使用例：**
- 開発サーバーの起動
- ビルドプロセスの実行
- テストスイートの並列実行

**利点：**
Claude Codeが他の作業を進めている間も、必要なプロセスをアクティブな状態で保持できます。

## 拡張性とカスタマイズ

### スラッシュコマンド (Slash Commands)

**概要：** 頻繁に使用する操作のカスタムショートカットです。

**使い方：**
- `.claude/commands/`フォルダにMarkdownファイルを配置
- `/`と入力するとメニューに表示
- `$ARGUMENTS`キーワードでパラメータを渡せる

**例：**
```markdown
<!-- .claude/commands/test.md -->
現在のプロジェクトのテストを実行してください
```

これにより`/test`コマンドが使用可能になります。

### MCP (Model Context Protocol) サーバー

**概要：** 外部ツールやデータソースに接続するためのオープンソース標準規格です。

**特徴：**
- 数百のツールやデータソースに接続可能
- Claude Codeにデータベース、API、その他のツールへのアクセスを提供
- MCPサーバーが公開するプロンプトはスラッシュコマンドとして利用可能

**利点：**
AI-ツール統合のための統一された標準インターフェースを提供します。

### スキル (Skills)

**概要：** 自動的にアクティブ化されるタスク固有の機能です。

**特徴：**
- `SKILL.md`ディスクリプタとオプションのスクリプトを含むフォルダ
- タスクコンテキストに一致すると自動的にアクティブ化
- スラッシュコマンドとは異なり、明示的な呼び出しが不要

**使用例：**
プロジェクト固有の開発パターンやベストプラクティスを自動適用できます。

### プラグイン (Plugins)

**概要：** スラッシュコマンド、サブエージェント、MCPサーバー、フックの組み合わせをパッケージ化して共有できる軽量な仕組みです。

**特徴：**
- `/plugin`コマンドで簡単にインストール
- ターミナルとVS Codeの両方で動作
- コミュニティ共有が可能

**インストール例：**
```bash
/plugin install [plugin-name]
```

## Claude Agent SDK

### 概要

Claude Agent SDKは、Claude Codeを支えるインフラストラクチャであり、2025年に開発者向けに公開されました。以前は「Claude Code SDK」と呼ばれていましたが、コーディング以外の幅広い機能を反映して「Claude Agent SDK」に改名されました。

### 主要機能

#### 1. ツールエコシステム

- ファイル操作
- コード実行
- Web検索
- MCP拡張性
- Bashコマンド実行
- CSVファイル読み込み
- ビジュアライゼーション構築
- メトリクス解釈

#### 2. コンテキスト管理

**自動コンパクト機能：**
コンテキスト制限に近づくと、以前のメッセージを自動的に要約し、エージェントがコンテキスト不足に陥らないようにします。

#### 3. 高度な権限管理

- エージェント機能の細かい制御
- ツールアクセスの正確な指定
- セキュリティとコンプライアンスの確保

#### 4. プロダクション対応機能

- 組み込みのエラーハンドリング
- セッション管理
- モニタリング機能
- 本番環境デプロイメント対応

### 対応言語

- **Python版**
- **TypeScript版**

両バージョンが利用可能で、異なる開発環境に対応しています。

## ベストプラクティス

### 1. チェックポイントの活用

大きな変更を試す前に、意図的にチェックポイントを作成することで、安全な実験環境を構築できます。

### 2. サブエージェントの戦略的利用

独立したタスク（フロントエンド/バックエンド、テスト/実装など）を並行して処理することで、開発速度を向上させられます。

### 3. フックによる品質保証

自動テスト、リント、フォーマットをフックに組み込むことで、一貫した品質を維持できます。

### 4. プラグインエコシステムの活用

コミュニティが作成したプラグインを活用することで、開発環境をすぐに強化できます。

### 5. MCPサーバーの統合

プロジェクト固有のデータベースやAPIをMCPサーバー経由で統合することで、Claude Codeの能力を拡張できます。

## VS Code vs ターミナル：どちらを選ぶべきか

### VS Code拡張機能が適している場合：

- ビジュアルなフィードバックを好む
- IDEに慣れている
- リアルタイムで変更を確認したい
- ファイルや画像を頻繁に添付する

### ターミナルインターフェースが適している場合：

- コマンドラインワークフローに慣れている
- 軽量な環境を好む
- スクリプト化や自動化を重視
- サブエージェントを頻繁に設定・変更する

## 実際の使用例

### 例1：フルスタック開発

```bash
# メインエージェント: Reactフロントエンドを構築
# サブエージェント1: Node.js APIを構築
# サブエージェント2: データベーススキーマを設計
# フック: コード変更後に自動的にテストを実行
```

### 例2：品質保証ワークフロー

```bash
# フック1: ファイル編集後に自動フォーマット
# フック2: コミット前にリント実行
# フック3: プッシュ前にテストスイート実行
# チェックポイント: 各主要機能実装後に自動保存
```

### 例3：AIデータ分析プロジェクト

```bash
# MCPサーバー: データベース接続
# スキル: データクリーニングパターン
# サブエージェント: ビジュアライゼーション生成
# バックグラウンドタスク: モデルトレーニング
```

## まとめ

Claude Code 2.0は、AI駆動開発ツールにおける大きな進歩を表しています。チェックポイント、サブエージェント、VS Code統合、フック、プラグインシステムなどの機能により、開発者はより効率的かつ安心してAIを活用できるようになりました。

2. **サブエージェント** - 並列開発による生産性の向上
3. **VS Code統合** - IDE内での直接的なAI支援
4. **拡張性** - プラグイン、MCP、フックによる高いカスタマイズ性
5. **Claude Agent SDK** - 独自のAIエージェント構築が可能

Claude Codeは、単なるコーディングアシスタントを超えて、包括的な開発パートナーへと進化を続けています。
